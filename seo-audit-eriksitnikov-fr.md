### 7. No canonical tags on any page

Every single page — homepage, /services, /services/seo-local, all others — is missing a canonical tag. This creates duplicate content risk if your site is accessible via www/non-www or http/https variants.

**Fix:** Add `alternates.canonical` to every page's metadata export:


## 🚀 PERFORMANCE & CODE

### Image optimization
- Your hero image (`/hero_bg_phone.webp`) already uses WebP — good.
- It's marked `loading="eager"` which is correct for above-the-fold LCP.
- Add explicit `width={540}` and `height={960}` to prevent layout shift (CLS).
- Use Next.js `<Image>` component if not already (it handles lazy loading, srcset, and format negotiation automatically).

### CSS blur effects
You have multiple decorative `div` elements with `filter: blur(100px)` and `blur(80px)`. These are GPU-accelerated in modern browsers but can cause paint issues on older mobile devices. Add `will-change: transform` to these elements or use `transform: translateZ(0)` to promote them to their own layer:

```tsx
<div
  aria-hidden="true"
  className="pointer-events-none absolute ..."
  style={{ filter: 'blur(100px)', willChange: 'transform' }}
/>
```

### Font loading
You're loading a variable font (Geist). Ensure it includes `font-display: swap` in the `@font-face` declaration to prevent invisible text during load. In Next.js, use the `next/font` module (which handles this automatically):

```ts
import { Geist } from 'next/font/google'
const geist = Geist({ subsets: ['latin'], display: 'swap' })
```
